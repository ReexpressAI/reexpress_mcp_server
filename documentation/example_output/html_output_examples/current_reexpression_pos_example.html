Reference [2, internal knowledge; Python file I/O semantics]: Iterating over a file object opened in text mode yields one line at a time (including the trailing newline character) and reads lazily, which is memory-efficient for large files. `str.strip()` removes leading/trailing whitespace including newline characters, so an empty/blank line strips to `""`, which is falsy and can be skipped. Reference [3, internal knowledge; JSONL/NDJSON format convention]: The JSON Lines format stores one JSON value (typically an object) per line, with `\n` (or `\r\n`) line separators. Answer: ```python import json def read_json_lines(filename): """Read a JSONL file and return a list of dicts, one per non-empty line.""" results = [] with open(filename, 'r', encoding='utf-8') as f: for line in f: line = line.strip() if line: results.append(json.loads(line)) return results ``` Notes: - `line.strip()` handles trailing newlines and skips blank or whitespace-only lines (satisfying the "non-empty line" requirement). - Iterating over the file object (`for line in f`) reads lazily, so this works on large files without loading everything into memory. - The `with` statement ensures the file is closed properly even if an exception is raised. - If tolerance for malformed lines is desired, wrap `json.loads(line)` in `try` / `except json.JSONDecodeError`.
